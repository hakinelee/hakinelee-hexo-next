网页设计最佳实践核对清单

发表于 2023-02-25 更新于 2023-03-12 分类于 开发必备 阅读次数： 本文字数： 1.6k 阅读时长 ≈ 6 分钟

好的网站不应该由客观好的设计来定义。同样重要的是网站的可用性、导航的便利性和可访问性。有了这些网页设计最佳实践，您就拥有了创建外观和功能良好的东西所需的一切。

请记住，这些是网页设计的最佳实践。根据您网站的性质，您可能无法完全关注每一项。但在你打破规则之前，至少了解它们存在的原因是有帮助的。

页面布局

  1. 吸引目标受众

  2. 统一网站header/logo

  3. 统一导航区域

  4. 让人一目了然的网页标题(title)，包括公司. 组织. 网站的名称

  5. 页脚区域——版权信息. 上一次更新日期. 联系人的邮件地址

  6. 良好的运用基本设计原则：重复. 对比. 近似和对齐

  7. 在1024*768或更高分辨率下显示时不需要水平滚动

  8. 页面中的文本. 图片. 空白均匀分布

  9. 文字和背景之间有良好的对比

  10. 在1024*768分辨率下，重复信息(标题header. logo和导航区)占据的区域不超过浏览器窗口的1/4~1/3

  11. 在1024x768分辨率下，主页“第一屏”(向下滚动之前)包含吸引人的. 有趣的信息

  12. 使用拨号连接时，主页在10秒钟之内下载完毕

  13. 视口元标签用于增强在智能手机上的显示效果

  14. 媒体查询为智能手机和平板电脑显示配置响应式的页面布局

导航

  1. 主导航链接标签清晰且统一

  2. 目标受众能够方便地使用导航功能

  3. 如果主导航区域使用了图片和多媒体，应在页脚提供清晰的文本链接(无障碍设计)

  4. 导航的结构为无序列表

  5. 提供导航协助，比如站点地图. “跳至内容”链接或者面包屑路径

  6. 所有的导航超链接都 “有效”–没有中断

颜色和图片

  1. 在页面背景/文本中使用最多三四种颜色

  2. 颜色的使用要和谐一致

  3. 文本颜色具有良好对比度

  4. 不要单独靠颜色来表达信息(无障碍设计)

  5. 颜色和图片的使用能够改善网站，而不是分散访问者的注意力

  6. 图片经过优化，不会明显拖慢下载速度

  7. 使用的每张图片都有清楚的目的

  8. img标记要用alt属性设置在浏览器或用户代理不支持图片的情况下显示的替代文本（无障碍设计）

  9. 动画图像不会使访问者分散注意力，要么不重复播放，要么只重复几次就可以了

多媒体

  1. 使用的每个音频/视频/Flash文件都目的明确

  2. 使用的音频/视频/Flash文件能够改善网站，而不是分散访问者的注意力

  3. 为使用的每个音频或视频文件提供文字旁白，即caption(无障碍设计)

  4. 标示音频或视频文件的下载时间

  5. 提供多媒体插件的下载地址

内容表示

  1. 使用常用字体，如Arial或TimesNewRoman。中文使用宋体或微软雅黑

  2. 使用的网络字体不超过一种

  3. 合理运用Web写作技术，包括标题. 项目列表. 短段落和短句. 空白等

  4. 统一字体. 字号和字体颜色

  5. 网页内容提供有意义和有用的信息

  6. 使用统一的方式组织内容

  7. 信息查找容易(最少点击)

  8. 要提示时间:上一次修订和版权日期要准确

  9. 内容不包括过时的材料

  10. 页面内容没有排版或语法错误

  11. 内容提供其他有用网站的链接

  12. 添加超链接文本时，避免“点击这里”的说法

  13. 如果没有使用标准链接颜色，必须统一设置一套颜色以表明链接的己访问/未访问状态

  14. 如果使用了图片和/或多媒体，同时提供与该内容等价的替代文字(无障碍设计)

功能

  1. 所有内部链接都正常工作

  2. 所有外部链接都正常工作

  3. 所有表单能像预期的那样工作

  4. 网页不报错

其他无障碍设计

  1. 如果主导航使用图片，文本链接在页面的页脚部分

  2. 如果主导航使用Flash，文本链接就在页面的页脚部分

  3. 导航的结构为无序列表

  4. 使用导航辅助工具，如网站地图. 跳过导航链接或面包屑。

  5. 颜色不是单独用来表达意思的

  6. 文字颜色与背景颜色有足够的对比

  7. 图片元素使用alt属性来配置一个备用的文本描述

  8. 如果使用图形来表达意思，则提供替代的文本等价物。

  9. 如果使用媒体来表达意思，则提供替代的文本等价物。

  10. 适当时使用旨在提高可访问性的属性，如标题。

  11. 为每个使用的音频或视频文件提供字幕

  12. html元素的lang属性表示页面的口语。

  13. 使用id和headers属性来提高表格数据的可及性

  14. 配置带有框架标题的框架，并在无框架区域放置有意义的内容

  15. 在恰当的地方使用专为改善无障碍访问而提供的属性，例如 alt 和 title

  16. role属性表示ARIA的标志性角色

  17. 为了帮助屏幕朗读器，html元素的lang属性要指明网页的朗读语言容性

浏览器兼容性

  1. 在主流版本的Internet Explorer(9或以上)中正常显示

  2. 在主流版本的Firefox中正常显示

  3. 在主流版本的Opera中正常显示

  4. 在主流版本的Safari(Mac和Windows)中正常显示

  5. 在主流版本的谷歌浏览器(GoogleChrome)中正常显示

  6. 在主流平板和智能手机上正常显示
